## Authentication Overview

Waves API uses **Bearer token authentication**. All API requests require your API key to be included in the `Authorization` header. Your API key authenticates your requests and links them to your Waves account for billing and usage tracking.

## Getting Your API Key

### Step 1: Sign Up or Log In

Visit [waves.smallest.ai](https://waves.smallest.ai) and create an account or log in to your existing account.

### Step 2: Generate API Key

1. Navigate to **API Key** tab in your account dashboard (left panel)

2. Click **Create a new API Key**

3. Copy your API key immediately (you won't be able to see it again)

4. Store it securely - treat it like a password

**Security Best Practice**: Store your API key as an environment variable named `SMALLEST_API_KEY` rather than hardcoding it in your application.

```bash
# Linux/Mac
export SMALLEST_API_KEY="your_api_key_here"

# Windows (Command Prompt)
set SMALLEST_API_KEY=your_api_key_here

# Windows (PowerShell)
$env:SMALLEST_API_KEY="your_api_key_here"
```

## Making Your First API Call

Let's make your first request to convert text to speech using the Lightning model.

### Endpoint

```
POST https://waves-api.smallest.ai/api/v1/lightning/get_speech
```

### Required Headers

```http
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

## Complete Examples

### cURL

```bash
curl -X POST https://waves-api.smallest.ai/api/v1/lightning/get_speech \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \

    "voice_id": "emily",
```

### Python

```python
import requests
import os

# Get API key from environment variable
api_key = os.environ.get("SMALLEST_API_KEY")

url = "https://waves-api.smallest.ai/api/v1/lightning/get_speech"
headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

payload = {
    "voice_id": "emily",
    "text": "Hello! This is my first API call with Waves.",
    "sample_rate": 24000,
    "add_wav_header": True
}

response = requests.post(url, json=payload, headers=headers)

if response.status_code == 200:
    with open("first_audio.wav", "wb") as f:
        f.write(response.content)
    print("âœ“ Success! Audio saved to first_audio.wav")
else:
    print(f"âœ— Error {response.status_code}: {response.text}")
```

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `voice_id` | string | Yes | Voice identifier (e.g., "emily", "jasmine", "arman") |
| `text` | string | Yes | Text to convert to speech |
| `sample_rate` | integer | No | Audio quality: `8000`, `16000`, or `24000` Hz (default: `24000`) |
| `add_wav_header` | boolean | No | Add WAV header for easier playback (default: `false`) |

## Response

**Success (200 OK)**

```
Content-Type: audio/wav
Body: [Binary audio data]
```

**Authentication Error (401 Unauthorized)**

```json
{
  "error": "Invalid or missing API key"
}
```

**Validation Error (400 Bad Request)**

```json
{
  "error": "Invalid request: missing required field 'text'"
}
```

**Rate Limit Error (429 Too Many Requests)**

```json
{
  "error": "Rate limit exceeded. Please try again later."
}
```

## Troubleshooting

### Common Issues

**"Invalid or missing API key"**

- Verify your API key is correct and properly formatted
- Ensure the `Authorization` header includes `Bearer` prefix
- Check that your API key hasn't been revoked

**"Rate limit exceeded"**

- You've sent too many requests in a short time
- Implement exponential backoff and retry logic
- Contact support if you need higher rate limits

**"Invalid voice_id"**

- Use the Get Voices endpoint to fetch available voice IDs: `GET https://waves-api.smallest.ai/api/v1/voices`
- Ensure the voice_id exists and is spelled correctly
